Module 0: The ETW Architecture (Win 10/11/12)

Welcome to the most comprehensive Event Viewer guide in existence. To understand detection, you must understand the source: Event Tracing for Windows (ETW). Whether you are hunting on Windows 10, Windows 11, or preparing for Windows 12, the underlying NT Kernel logs data through ETW providers.

Native Event Viewer is powerful, but it is blind to deep C++ memory hooks and external API calls. To achieve 1200% visibility and the 100% detection rate you require, you must pair native Windows logs with Microsoft Sysmon (System Monitor).

The Golden Prerequisites

If these features are not enabled via Local Group Policy Editor (gpedit.msc), you will miss the malware entirely.

  • Audit Process Creation: Enable Computer Configuration > Windows Settings > Security Settings > Advanced Audit Policy Configuration > Detailed Tracking > Audit Process Creation (Success and Failure).
  • Include Command Line in Process Creation Events: Enable Computer Configuration > Administrative Templates > System > Audit Process Creation. Without this, you only see that cmd.exe ran, not the malicious string it executed.
  • PowerShell Script Block Logging: Enable Computer Configuration > Administrative Templates > Windows Components > Windows PowerShell > Turn on PowerShell Script Block Logging.
  • Install Sysmon: Download from Sysinternals, use a robust config (like SwiftOnSecurity), and run sysmon.exe -accepteula -i sysmonconfig.xml.

Module 1: Anti-Forensics, Timestomping & Evasion

Advanced malware, sophisticated cheats, and human operators will actively attempt to blind your telemetry before executing their primary payload.

Event ID 1102 Event ID 104

Audit Logs Wiped

Windows Logs > Security (1102) & System (104)

What it is: The system or security event log was explicitly cleared.

Why it's SUS (Suspicious): Normal users and standard IT operations almost never clear the security log manually. If 1102 fires, you are actively breached or a rootkit is attempting to hide its installation trail.

Deep Analysis Steps:

  • Open the event and look at the Subject: Account Name and Account Domain. This tells you which user context was compromised to issue the clear command.
  • Immediately pivot to Sysmon or EDR telemetry to look for tools like wevtutil cl System or PowerShell commands like Clear-EventLog executing just prior to the wipe.
Event ID 4616

System Time Manipulation

Windows Logs > Security

What it is: The system clock was changed.

Why it's SUS: Known as "Timestomping." Advanced attackers roll back the system clock to a date when a leaked, malicious code-signing certificate was still valid. This allows them to load a Ring-0 kernel driver without triggering Driver Signature Enforcement (DSE).

Hunt Rule: Filter out `SubjectLogonId` for `LOCAL SERVICE` (which is standard NTP time sync). If a user account or an unknown executable initiates the time change, investigate immediately.

Event ID 4688

USN Journal Manipulation

Windows Logs > Security

What it is: Process Creation log. You are hunting for specific command-line arguments targeting the NTFS file system tracking.

Why it's SUS: The Update Sequence Number (USN) journal tracks every file dropped or deleted on the drive. Malware clears this to prevent forensic analysts from finding the payload origin.

Hunt Rule: Filter ID 4688 and search the Process Command Line for: fsutil usn deletejournal /D C:

Module 2: USB, PCI, and DMA Hardware Attacks

Physical vectors bypass software firewalls completely. This includes malicious USB drops (Rubber Duckies) and PCIe screamer cards used by high-end espionage and game cheats to read physical RAM via Direct Memory Access (DMA).

Event ID 2003 / 2100 Event ID 400 / 410

Reliable USB / Device Plug & Unplug

App & Services > Microsoft > Windows > DriverFrameworks-UserMode > Operational (2003/2100)
App & Services > Microsoft > Windows > Kernel-PnP > Configuration (400/410)

What it is: The most reliable way to track device connections. 2003/400 means plugged in/configured. 2100/410 means unplugged/removed.

Why it's SUS: You are looking for mismatched hardware profiles. If a user inserts a "USB Flash Drive", but Event Viewer registers a new Human Interface Device (HID) keyboard that immediately executes 500 keystrokes, you have found a malicious payload delivery device.

Deep Analysis Steps:

  • Look at the Instance ID in the log. It will contain the Vendor ID (VID) and Product ID (PID), e.g., USB\VID_045E&PID_07B2.
  • Search that VID/PID online. If the log says it's a storage device but the VID maps to a generic keyboard controller, it's forged hardware.
Event ID 4122 / 4123

Unprotected DMA Device (PCIe Screamers)

App & Services > Microsoft > Windows > BitLocker-API > Management

What it is: Windows detected a device plugged into a high-speed bus (PCIe/Thunderbolt) that is capable of Direct Memory Access (DMA), but the system's IOMMU (Kernel DMA Protection) failed to protect the memory.

Why it's SUS: DMA cards are external PCI devices that read/write system RAM directly, bypassing the CPU and OS entirely. This is heavily used by custom firmware cheats and advanced rootkits.

Hunt Rule: Custom DMA cards often try to spoof legitimate devices. If you see ID 4122 triggered by a device claiming to be a generic "Intel Wi-Fi Adapter" that was just plugged into a PCIe slot, physical memory is being scraped.

Module 3: UEFI Bootkits, Secure Boot & Database Auditing

If malware loads before the OS kernel starts (via the UEFI), it owns the entire system. It can patch the Windows loader in memory, bypassing all subsequent security measures.

Event ID 1801 / 1802

TPM-WMI: Secure Boot Certificate Updates

Windows Logs > System

What it is: Windows Update downloaded new Secure Boot databases (DB/DBX) to protect against UEFI bootkits like BlackLotus. ID 1801 means the certificates are pending; the firmware has not accepted them into NVRAM yet.

The Absolute Truth Test (PowerShell Validation):
Event Viewer only tells you the OS tried to update the keys. To know if you are actually protected, you must interrogate the UEFI memory directly. Open an Admin PowerShell and run: 
([System.Text.Encoding]::ASCII.GetString((Get-SecureBootUEFI db).bytes) -match 'Windows UEFI CA 2023')

Deciphering the PowerShell Output:

Output Meaning Security Posture
True The motherboard firmware successfully wrote the new 2023 Microsoft certificates to the physical NVRAM. Secure. The system will block compromised/leaked Windows bootloaders.
False The firmware rejected the update or requires a physical BIOS flash/factory key reset to accept it. Critically Vulnerable. The system is running the 2011 keys and is exposed to the BlackLotus bootkit and BYOVD downgrades.
Event ID 3033

Code Integrity Check Failure (Fake Signatures)

App & Services > Microsoft > Windows > CodeIntegrity > Operational

What it is: Windows Driver Signature Enforcement (DSE) blocked a file from loading because its cryptographic signature was invalid, revoked, or non-existent.

Why it's SUS: Attackers steal certificates from legitimate software companies to sign their malware. When those certs are discovered and revoked by Microsoft (placed in the DBX), ID 3033 fires when the malware tries to run.

Module 4: Kernel Drivers (Ring-0) & BYOVD

To execute devastating actions like hiding processes or unhooking EDR sensors, malware must elevate to Ring-0 (the Kernel).

Event ID 7045 Event ID 4697

New Service / Kernel Driver Installed

Windows Logs > System (7045) & Security (4697)

What it is: A new service, often a .sys kernel driver, was registered with the Windows Service Control Manager (SCM).

Bring Your Own Vulnerable Driver (BYOVD):
Attackers know unsigned drivers are blocked. Instead, they drop a legitimately signed, but severely outdated and vulnerable driver (e.g., an old version of `capcom.sys` or an Intel diagnostic tool). They install it (triggering 7045), then exploit it to map their own unsigned malicious code into kernel space.

Deep Analysis Steps:

  • Examine the ImagePath. Legitimate drivers live in C:\Windows\System32\drivers\.
  • If the ImagePath is C:\Windows\Temp\random.sys, C:\Users\Public\driver.sys, or hidden in an AppData subfolder, it is a massive red flag.
  • Check the Service Type. If it says kernel mode driver and the source is unknown, isolate the machine.

Module 5: Memory Injections & Thread Hijacking

This module requires Sysmon. Advanced threats do not write malware.exe to disk. They inject custom C++ payloads directly into the live memory of legitimate processes (like explorer.exe or a running video game to draw an ImGui overlay).

Sysmon ID 10

Process Access (Thread Suspension & Manipulation)

App & Services > Microsoft > Windows > Sysmon > Operational

What it is: One process requested a handle to open another process's memory space using the Windows OpenProcess API.

The Hexadecimal Access Mask Decorder Ring:

When you look at the event, find the GrantedAccess field. This hex code tells you exactly what the attacker's C++ code is trying to do to the victim process.

Access Mask (Hex) Windows API Constant Forensic Meaning / Malicious Use Case
0x1FFFFF PROCESS_ALL_ACCESS God Mode. The tool is trying to completely control the target. Often seen when Mimikatz targets lsass.exe to dump passwords.
0x0008 PROCESS_VM_OPERATION Memory Hooking. Required to use VirtualAllocEx. The attacker is allocating space inside the target process to write their payload.
0x0020 PROCESS_VM_WRITE Payload Delivery. The attacker is calling WriteProcessMemory to insert the malicious bytes into the allocated space.
0x0010 PROCESS_VM_READ Data Scraping. The tool is reading memory arrays. Commonly used by external cheats reading player coordinates from a game client matrix.
0x0800 PROCESS_SUSPEND_RESUME Thread Freezing. The attacker calls SuspendThread to freeze the process, uses SetThreadContext to point the execution to their injected payload, and calls ResumeThread.
Why it's SUS: Legitimate programs rarely need PROCESS_ALL_ACCESS or PROCESS_VM_WRITE handles to other running programs.
Sysmon ID 8

CreateRemoteThread (Code Execution)

What it is: Detects the final stage of injection. After allocating memory and writing the payload (ID 10), the attacker calls CreateRemoteThread to force the victim process to execute the malicious bytes.

Hunt Rule: Look at SourceImage vs TargetImage. If powershell.exe or a random background app (Source) creates a remote thread inside winlogon.exe or notepad.exe (Target), it is definitive proof of process hollowing or DLL injection.

Module 6: PowerShell & Fileless Execution

Living-off-the-Land (LotL) attacks use native system administration tools to execute code, avoiding the drop of an easily detectable `.exe` file.

Event ID 4104

PowerShell Script Block Logging

App & Services > Microsoft > Windows > PowerShell > Operational

What it is: Captures the full content of PowerShell code blocks as they are executed. Crucially, it captures the code after it has been unrolled and de-obfuscated in memory.

Why it's SUS: Fileless malware exists entirely as a giant encoded string passed directly to the shell. ID 4104 will reveal the true intentions of the script.

Keywords to Hunt For in the ScriptBlockText:

  • FromBase64String: The attacker is unpacking a hidden payload.
  • [System.Runtime.InteropServices.Marshal]::Copy: C# reflection used to copy bytes directly into memory.
  • VirtualAlloc or WriteProcessMemory: The PowerShell script is actively performing memory injection (see Module 5) without needing a compiled C++ `.exe`.
  • System.Reflection.Assembly.Load: Loading a malicious .NET DLL straight into RAM.
Event ID 4688

Command Line Obfuscation

Look at the process creation command line for extreme obfuscation tactics. E.g., cmd.exe /c "set x=powershell&& call %x% -enc JABz...". The use of carets (^), environmental variable slicing, or `-enc` (EncodedCommand) are hallmarks of an evasion attempt.

Module 7: Persistence & Scheduled Tasks

Once memory is injected or a rootkit is loaded, it must survive a system reboot. Attackers blend in with background noise.

Event ID 4698

Scheduled Task Created

Windows Logs > Security

What it is: A new task was registered with the Windows Task Scheduler.

Why it's SUS: Attackers create tasks that trigger "On Logon" or "On Idle". They will name the task something deceptive like WindowsUpdateCore or GoogleCrashHandler.

Deep Analysis Steps:

  • Open the event log and scroll down to the Event XML data.
  • Find the <Command> and <Arguments> tags.
  • If the task is named "AdobeUpdate" but the command is wscript.exe launching a .vbs file from C:\ProgramData\, it's persistence.
Event ID 5861

WMI Activity (Advanced Persistence)

App & Services > Microsoft > Windows > WMI-Activity > Operational

Advanced adversaries use Windows Management Instrumentation (WMI) Event Consumers. They write a script directly into the WMI repository that triggers automatically when a specific system event occurs (e.g., when a specific process starts or the system has been up for 5 minutes). This is incredibly stealthy as there are no files on disk and no scheduled tasks to find.

Module 8: Web Requests & C2 Beacons

Malware must phone home to its Command and Control (C2) server to receive instructions or exfiltrate data. Native logging struggles with process-to-network attribution, so we rely on Sysmon.

Sysmon ID 22 Sysmon ID 3

DNS Queries & Network Connections

App & Services > Microsoft > Windows > Sysmon > Operational

What it is: ID 22 tracks which process asked to resolve a domain name. ID 3 tracks the exact IP, port, and process that established an outbound connection.

Why it's SUS: This is how you catch Process Hollowing. If notepad.exe, calc.exe, or svchost.exe (running from the wrong directory) suddenly resolves a strange domain (ID 22) and opens a TCP port 443 connection to an unknown IP in Russia (ID 3), you have found your C2 beacon.

Hunt Rule: Look for "Beaconing" behavior. If a process initiates a Sysmon ID 3 connection to the exact same IP address exactly every 5 minutes and 30 seconds (with some jitter), that is an automated malware callback, not a human browsing the web.

Module 9: Crashed Files & Bad Pointers

Attackers make mistakes. Poorly written C++ malware, broken game cheats, or improperly configured ImGui overlays frequently fail when trying to hook into memory, resulting in target process crashes.

Event ID 1000 Event ID 1001

Application Error (AppCrash)

Windows Logs > Application

What it is: ID 1000 logs the exact faulting module that caused a process to unexpectedly terminate. ID 1001 logs the Windows Error Reporting (WER) bucket data.

Deciphering the Exception Codes:

Exception Code Technical Meaning Forensic Implication
0xc0000005 Access Violation The #1 indicator of a failed memory injection. A foreign `.dll` or injected thread attempted to read or write to a memory address it didn't have permission for, or the pointer was null/invalid. Common with broken overlays or outdated C++ hooks.
0xc0000374 Heap Corruption The injected payload mismanaged memory allocation, overwriting critical data structures in the target process.
0xc0000409 Stack Buffer Overrun Often seen during active exploitation attempts where an attacker sends a payload too large for the buffer, triggering stack protection (GS cookies) which safely kills the app.
The Smoking Gun:
Look at the Faulting Module Name. If an application like Discord.exe crashes (0xc0000005), and the Faulting Module Name is an unknown `.dll` file sitting in C:\Users\Name\AppData\Local\Temp\hook.dll, you have just found the exact malicious payload that attempted (and failed) to inject itself.
